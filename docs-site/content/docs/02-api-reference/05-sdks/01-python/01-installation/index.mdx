# Installation

The TraceRTM Python SDK can be installed using pip, Poetry, or other Python package managers. This guide covers all installation methods and configuration options.

## Requirements

- **Python**: 3.9 or higher
- **Dependencies**: Automatically installed
  - `httpx` - HTTP client
  - `pydantic` - Data validation

## Quick Install

### Using pip

```bash
pip install tracertm
```

### Using Poetry

```bash
poetry add tracertm
```

### Using pipenv

```bash
pipenv install tracertm
```

### Using uv

```bash
uv add tracertm
```

---

## Version Selection

### Install Specific Version

```bash
pip install tracertm==2.4.0
```

### Install Latest Pre-release

```bash
pip install tracertm --pre
```

### Install from Git

```bash
pip install git+https://github.com/tracertm/tracertm-python.git@main
```

---

## Optional Dependencies

### Async Support

The SDK includes async support by default, but you can install with extra async utilities:

```bash
pip install tracertm[async]
```

### Development Dependencies

For testing and development:

```bash
pip install tracertm[dev]
```

This includes:
- `pytest` - Testing framework
- `pytest-asyncio` - Async test support
- `respx` - HTTP mocking
- `mypy` - Type checking

### All Extras

```bash
pip install tracertm[all]
```

## Virtual Environment Setup

### Using venv

```bash
# Create virtual environment
python -m venv .venv

# Activate (Linux/macOS)
source .venv/bin/activate

# Activate (Windows)
.venv\Scripts\activate

# Install SDK
pip install tracertm
```

### Using Poetry

```bash
# Create new project with Poetry
poetry new my-tracertm-project
cd my-tracertm-project

# Add SDK
poetry add tracertm

# Activate shell
poetry shell
```

### Using pipenv

```bash
# Create Pipfile and install
pipenv install tracertm

# Activate shell
pipenv shell
```

## Configuration

### Environment Variables

Set your API key as an environment variable:

```bash
# Linux/macOS
export TRACERTM_API_KEY="rtm_live_sk_..."

# Windows (PowerShell)
$env:TRACERTM_API_KEY = "rtm_live_sk_..."

# Windows (CMD)
set TRACERTM_API_KEY=rtm_live_sk_...
```

### Configuration File

Create a `.env` file in your project root:

```bash
TRACERTM_API_KEY=rtm_live_sk_...
TRACERTM_BASE_URL=https://api.tracertm.dev/v1
TRACERTM_TIMEOUT=30
```

Use with `python-dotenv`:

```python
from dotenv import load_dotenv
from tracertm import TraceRTMClient

load_dotenv()
client = TraceRTMClient()  # Reads from environment
```

## Proxy Configuration

### Environment Variables

```bash
export HTTP_PROXY="http://proxy.example.com:8080"
export HTTPS_PROXY="http://proxy.example.com:8080"
```

### Client Configuration

```python
from tracertm import TraceRTMClient

client = TraceRTMClient(
    api_key="rtm_live_sk_...",
    proxies={
        "http://": "http://proxy.example.com:8080",
        "https://": "http://proxy.example.com:8080"
    }
)
```

## SSL/TLS Configuration

### Custom CA Bundle

```python
from tracertm import TraceRTMClient

client = TraceRTMClient(
    api_key="rtm_live_sk_...",
    verify="/path/to/ca-bundle.crt"
)
```

### Disable SSL Verification (Not Recommended)

```python
# Only for development/testing
client = TraceRTMClient(
    api_key="rtm_live_sk_...",
    verify=False
)
```

## Troubleshooting

### Import Error

```
ModuleNotFoundError: No module named 'tracertm'
```

**Solution**: Ensure you've activated your virtual environment and installed the package:

```bash
pip install tracertm
```

### SSL Certificate Error

```
ssl.SSLCertVerificationError: certificate verify failed
```

**Solution**: Update certificates:

```bash
pip install --upgrade certifi
```

Or specify a CA bundle:

```python
client = TraceRTMClient(
    api_key="rtm_live_sk_...",
    verify="/path/to/ca-bundle.crt"
)
```

### Version Conflict

```
ERROR: tracertm requires httpx>=0.24.0, but you have httpx 0.23.0
```

**Solution**: Upgrade dependencies:

```bash
pip install --upgrade tracertm httpx
```
