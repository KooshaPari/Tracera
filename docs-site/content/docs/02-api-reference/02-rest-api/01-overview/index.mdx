# REST API Overview

The TraceRTM REST API provides comprehensive access to all platform features including projects, items, links, workflows, and more. This guide covers API fundamentals, conventions, and best practices.

## API Basics

### Base URL

```
https://api.tracertm.dev/v1
```

All API endpoints are relative to this base URL.

### Request Format

The API accepts and returns JSON. Include the `Content-Type` header for requests with bodies:

```bash
curl -X POST "https://api.tracertm.dev/v1/projects" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "My Project", "key": "PROJ"}'
```

### Response Format

All responses follow a consistent structure:

```json
{
  "data": { ... },
  "meta": {
    "request_id": "req_abc123xyz"
  }
}
```

List responses include pagination:

```json
{
  "data": [ ... ],
  "meta": {
    "total": 245,
    "page": 1,
    "per_page": 20,
    "total_pages": 13
  },
  "links": {
    "self": "https://api.tracertm.dev/v1/projects?page=1",
    "next": "https://api.tracertm.dev/v1/projects?page=2",
    "last": "https://api.tracertm.dev/v1/projects?page=13"
  }
}
```

## Core Resources

### Projects

Projects are containers for items, links, and workflows.

```http
GET    /v1/projects                 # List projects
POST   /v1/projects                 # Create project
GET    /v1/projects/{id}            # Get project
PATCH  /v1/projects/{id}            # Update project
DELETE /v1/projects/{id}            # Delete project
```

### Items

Items represent requirements, test cases, bugs, and other trackable entities.

```http
GET    /v1/projects/{project_id}/items          # List items
POST   /v1/projects/{project_id}/items          # Create item
GET    /v1/items/{id}                           # Get item
PATCH  /v1/items/{id}                           # Update item
DELETE /v1/items/{id}                           # Delete item
POST   /v1/items/{id}/transition                # Change status
```

### Links

Links create traceability relationships between items.

```http
GET    /v1/projects/{project_id}/links          # List links
POST   /v1/links                                # Create link
GET    /v1/links/{id}                           # Get link
DELETE /v1/links/{id}                           # Delete link
```

### Workflows

Workflows define item lifecycles and status transitions.

```http
GET    /v1/projects/{project_id}/workflows      # List workflows
POST   /v1/projects/{project_id}/workflows      # Create workflow
GET    /v1/workflows/{id}                       # Get workflow
PATCH  /v1/workflows/{id}                       # Update workflow
```

## HTTP Methods

| Method | Description | Idempotent |
|--------|-------------|------------|
| `GET` | Retrieve resources | Yes |
| `POST` | Create resources | No |
| `PATCH` | Partial update | Yes |
| `PUT` | Full replacement | Yes |
| `DELETE` | Delete resources | Yes |

## Common Parameters

### Pagination

```bash
# Offset-based pagination
?page=2&per_page=50

# Cursor-based pagination
?cursor=eyJpZCI6MTAwfQ&limit=50
```

### Filtering

```bash
# Simple equality
?status=approved

# Multiple values
?status=approved,review

# Operators
?created_at[gte]=2024-01-01
```

### Sorting

```bash
# Ascending
?sort=created_at

# Descending
?sort=-created_at

# Multiple fields
?sort=-priority,created_at
```

### Field Selection

```bash
# Return only specific fields
?fields=id,title,status

# Include related resources
?include=links,comments
```

## Response Codes

### Success Codes

| Code | Description | When Used |
|------|-------------|-----------|
| `200` | OK | Successful GET, PATCH |
| `201` | Created | Successful POST |
| `204` | No Content | Successful DELETE |

### Client Errors

| Code | Description | Common Cause |
|------|-------------|--------------|
| `400` | Bad Request | Invalid JSON, malformed request |
| `401` | Unauthorized | Invalid or missing authentication |
| `403` | Forbidden | Insufficient permissions |
| `404` | Not Found | Resource doesn't exist |
| `409` | Conflict | Duplicate or version conflict |
| `422` | Unprocessable Entity | Validation error |
| `429` | Too Many Requests | Rate limit exceeded |

### Server Errors

| Code | Description | Action |
|------|-------------|--------|
| `500` | Internal Server Error | Retry or contact support |
| `502` | Bad Gateway | Retry after delay |
| `503` | Service Unavailable | Check status page |
| `504` | Gateway Timeout | Retry with backoff |

---

## Error Format

All errors follow a consistent structure:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Request validation failed",
    "status": 422,
    "details": {
      "field": "title",
      "reason": "Title is required"
    },
    "request_id": "req_abc123xyz"
  }
}
```

## Rate Limiting

API requests are rate limited per account:

| Plan | Requests/Minute | Burst |
|------|-----------------|-------|
| Free | 60 | 10 |
| Pro | 300 | 50 |
| Team | 600 | 100 |
| Enterprise | 1,200 | 200 |

Rate limit headers are included in all responses:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 287
X-RateLimit-Reset: 1704067260
```

See [Rate Limiting](/docs/api-reference/rest-api/rate-limiting) for details.

## Versioning

The API uses URL versioning. The current version is `v1`.

```
https://api.tracertm.dev/v1/projects
```

Breaking changes result in new versions. Non-breaking changes are added to the current version.

## SDK Support

Official SDKs are available for major languages:

### Python

```python

client = TraceRTMClient(api_key="rtm_live_sk_...")

# List projects
projects = client.projects.list()

# Create item
```

## Best Practices

### Use Pagination

Always paginate list requests to avoid large responses:

```python
# Good: Use pagination
for page in client.projects.list_pages(per_page=50):
    process_projects(page.data)

# Avoid: Fetching all at once for large datasets
all_projects = client.projects.list(per_page=10000)  # May timeout
```

### Handle Errors

Implement proper error handling:

```python
from tracertm.exceptions import NotFoundError, RateLimitError

try:
    item = client.items.get("item_123")
except NotFoundError:
    print("Item not found")
except RateLimitError as e:
    time.sleep(e.retry_after)
    # Retry
```

### Use Idempotency Keys

For critical operations, use idempotency keys:

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_123/items" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Idempotency-Key: unique-request-id-123" \
  -d '{"type": "requirement", "title": "..."}'
```

### Cache Appropriately

Cache responses to reduce API calls:

```python
from functools import lru_cache

@lru_cache(maxsize=100)
def get_project(project_id: str):
    return client.projects.get(project_id)
```

### Use Webhooks for Events

Instead of polling, use webhooks for real-time updates:

```python
webhook = client.webhooks.create(
    url="https://your-app.com/webhook",
    events=["item.created", "item.updated"]
)
```
