# Batch Operations

Batch operations allow you to execute multiple API requests in a single HTTP call. This reduces network overhead, enables atomic transactions, and improves performance for bulk data operations.

## Quick Reference

```bash
# Single batch request
POST /v1/batch

# Project-scoped batch
POST /v1/projects/{project_id}/batch
```

---

## Batch Request Format

### Request Structure

```json
{
  "atomic": true,
  "operations": [
    {
      "id": "op1",
      "method": "POST",
      "path": "/v1/projects/proj_abc123/items",
      "body": {
        "type": "requirement",
        "title": "First requirement"
      }
    },
    {
      "id": "op2",
      "method": "POST",
      "path": "/v1/projects/proj_abc123/items",
      "body": {
        "type": "requirement",
        "title": "Second requirement"
      },
      "depends_on": ["op1"]
    }
  ]
}
```

### Operation Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique operation identifier |
| `method` | string | Yes | HTTP method: `GET`, `POST`, `PATCH`, `DELETE` |
| `path` | string | Yes | API endpoint path |
| `body` | object | Conditional | Request body (for POST/PATCH) |
| `headers` | object | No | Additional headers for this operation |
| `depends_on` | array | No | Operation IDs this depends on |

### Batch Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `atomic` | boolean | false | Roll back all on any failure |
| `stop_on_error` | boolean | false | Stop processing on first error |
| `continue_on_error` | boolean | true | Process all regardless of errors |
| `parallel` | boolean | false | Execute independent ops in parallel |

## Execute Batch

```http
POST /v1/batch
```

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/batch" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "atomic": true,
    "operations": [
      {
        "id": "create_req_1",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {
          "type": "requirement",
          "title": "User Authentication",
          "description": "System shall authenticate users via OAuth 2.0",
          "priority": "critical"
        }
      },
      {
        "id": "create_req_2",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {
          "type": "requirement",
          "title": "Session Management",
          "description": "System shall manage user sessions",
          "priority": "high"
        }
      },
      {
        "id": "create_link",
        "method": "POST",
        "path": "/v1/links",
        "body": {
          "source_id": "${create_req_1.data.id}",
          "target_id": "${create_req_2.data.id}",
          "type": "depends_on"
        },
        "depends_on": ["create_req_1", "create_req_2"]
      }
    ]
  }'
```

## Variable References

Use results from previous operations in dependent operations:

### Syntax

```
${operation_id.path.to.value}
```

### Examples

```json
{
  "operations": [
    {
      "id": "create_parent",
      "method": "POST",
      "path": "/v1/projects/proj_abc123/items",
      "body": {"type": "requirement", "title": "Parent"}
    },
    {
      "id": "create_child",
      "method": "POST",
      "path": "/v1/projects/proj_abc123/items",
      "body": {
        "type": "requirement",
        "title": "Child",
        "parent_id": "${create_parent.data.id}"
      },
      "depends_on": ["create_parent"]
    },
    {
      "id": "update_parent",
      "method": "PATCH",
      "path": "/v1/items/${create_parent.data.id}",
      "body": {
        "metadata": {
          "child_count": 1,
          "first_child": "${create_child.data.item_number}"
        }
      },
      "depends_on": ["create_child"]
    }
  ]
}
```

## Project-Scoped Batch

Execute batch operations within a specific project context:

```http
POST /v1/projects/{project_id}/batch
```

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_abc123/batch" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "atomic": true,
    "operations": [
      {
        "id": "item1",
        "method": "POST",
        "path": "/items",
        "body": {"type": "requirement", "title": "Req 1"}
      },
      {
        "id": "item2",
        "method": "POST",
        "path": "/items",
        "body": {"type": "test_case", "title": "Test 1"}
      },
      {
        "id": "link1",
        "method": "POST",
        "path": "/links",
        "body": {
          "source_id": "${item1.data.id}",
          "target_id": "${item2.data.id}",
          "type": "verified_by"
        },
        "depends_on": ["item1", "item2"]
      }
    ]
  }'
```

## SDK Examples

### Python

```python
from tracertm import TraceRTMClient

client = TraceRTMClient(api_key="rtm_live_sk_...")

# Simple batch
result = client.batch.execute([
    {
        "id": "req1",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {"type": "requirement", "title": "Requirement 1"}
    },
    {
        "id": "req2",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {"type": "requirement", "title": "Requirement 2"}
    }
], atomic=True)

print(f"Created {result.successful} items")

# With dependencies
result = client.batch.execute([
    {
        "id": "parent",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {"type": "requirement", "title": "Parent Requirement"}
    },
    {
        "id": "child",
        "method": "POST",
        "path": "/v1/projects/proj_abc123/items",
        "body": {
            "type": "requirement",
            "title": "Child Requirement",
            "parent_id": "${parent.data.id}"
        },
        "depends_on": ["parent"]
    },
    {
        "id": "link",
        "method": "POST",
        "path": "/v1/links",
        "body": {
            "source_id": "${parent.data.id}",
            "target_id": "${child.data.id}",
            "type": "derives_from"
        },
        "depends_on": ["parent", "child"]
    }
], atomic=True)

# Access results
for op_result in result.results:
    if op_result.success:
        print(f"{op_result.id}: Created {op_result.data.get('id')}")
    else:
        print(f"{op_result.id}: Failed - {op_result.error}")

# Batch helper for bulk creates
items = client.items.bulk_create(
    project_id="proj_abc123",
    items=[
        {"type": "requirement", "title": "Req 1"},
        {"type": "requirement", "title": "Req 2"},
        {"type": "requirement", "title": "Req 3"}
    ]
)

print(f"Created {len(items)} items")

# Batch helper for bulk updates
client.items.bulk_update([
    {"id": "item_001", "status": "approved"},
    {"id": "item_002", "status": "approved"},
    {"id": "item_003", "status": "approved"}
])

# Batch helper for bulk links
client.links.bulk_create([
    {"source_id": "item_req_001", "target_id": "item_tc_001", "type": "verified_by"},
    {"source_id": "item_req_002", "target_id": "item_tc_002", "type": "verified_by"},
    {"source_id": "item_req_003", "target_id": "item_tc_003", "type": "verified_by"}
])
```

## Atomicity Behavior

### `atomic: true`

All operations succeed or all are rolled back:

```json
{
  "atomic": true,
  "operations": [
    {"id": "op1", "method": "POST", "path": "...", "body": {...}},
    {"id": "op2", "method": "POST", "path": "...", "body": {...}},
    {"id": "op3", "method": "POST", "path": "...", "body": {...}}
  ]
}
```

If `op2` fails:
- `op1` changes are rolled back
- `op3` is not executed
- All data remains unchanged

### `atomic: false` (default)

Each operation is independent:

```json
{
  "atomic": false,
  "operations": [
    {"id": "op1", "method": "POST", "path": "...", "body": {...}},
    {"id": "op2", "method": "POST", "path": "...", "body": {...}},
    {"id": "op3", "method": "POST", "path": "...", "body": {...}}
  ]
}
```

If `op2` fails:
- `op1` changes persist
- `op3` still executes
- Results show individual success/failure

## Error Codes

### Batch Errors

| Code | Description |
|------|-------------|
| `BATCH_TOO_LARGE` | Too many operations in batch |
| `BATCH_TIMEOUT` | Batch exceeded timeout |
| `BATCH_ROLLBACK_FAILED` | Atomic rollback failed |
| `CIRCULAR_DEPENDENCY` | Circular depends_on reference |
| `INVALID_OPERATION_REFERENCE` | Invalid variable reference |
| `DEPENDENCY_FAILED` | Dependent operation failed |

## Best Practices

### 1. Use Atomic for Related Changes

```python
# Good: Related changes are atomic
result = client.batch.execute([
    {"id": "parent", "method": "POST", ...},
    {"id": "child", "method": "POST", ..., "depends_on": ["parent"]},
    {"id": "link", "method": "POST", ..., "depends_on": ["parent", "child"]}
], atomic=True)

# Bad: Related changes may be partially applied
result = client.batch.execute([...], atomic=False)
```

### 2. Batch Similar Operations

```python
# Good: Batch creates together
client.batch.execute([
    {"method": "POST", "path": "/items", "body": {...}},
    {"method": "POST", "path": "/items", "body": {...}},
    {"method": "POST", "path": "/items", "body": {...}}
])

# Bad: Individual requests
for item in items:
    client.items.create(**item)  # Multiple network calls
```

### 3. Handle Partial Failures

```python
result = client.batch.execute(operations, atomic=False)

if result.status == "partial":
    for op_result in result.results:
        if not op_result.success:
            logger.error(f"Operation {op_result.id} failed: {op_result.error}")
            # Handle or retry failed operation
```

### 4. Use Dependencies for Order

```python
# Good: Explicit dependencies
operations = [
    {"id": "req", "method": "POST", "path": "/items", ...},
    {"id": "test", "method": "POST", "path": "/items", ..., "depends_on": ["req"]},
    {"id": "link", "method": "POST", "path": "/links", ..., "depends_on": ["req", "test"]}
]

# Bad: Assuming order without dependencies
operations = [
    {"id": "req", "method": "POST", ...},
    {"id": "test", "method": "POST", ...},  # May execute before req
    {"id": "link", "method": "POST", ...}   # May fail if req/test not done
]
```
