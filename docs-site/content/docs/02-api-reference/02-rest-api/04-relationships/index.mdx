# Relationships API

Traceability links (relationships) connect items in TraceRTM, enabling impact analysis, coverage tracking, and compliance reporting. This guide covers creating, querying, and analyzing relationships.

## Endpoints Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/v1/projects/{project_id}/links` | Create relationship |
| `GET` | `/v1/projects/{project_id}/links` | List relationships |
| `GET` | `/v1/links/{link_id}` | Get relationship details |
| `PATCH` | `/v1/links/{link_id}` | Update relationship |
| `DELETE` | `/v1/links/{link_id}` | Delete relationship |
| `POST` | `/v1/projects/{project_id}/links/batch` | Batch create |
| `GET` | `/v1/items/{item_id}/relationships` | Get item relationships |
| `GET` | `/v1/projects/{project_id}/trace` | Trace paths |
| `GET` | `/v1/projects/{project_id}/impact` | Impact analysis |

## The Relationship Object

```json
{
  "id": "link_abc123xyz",
  "project_id": "proj_abc123",
  "source_id": "item_req_001",
  "target_id": "item_tc_001",
  "type": "verified_by",
  "direction": "forward",
  "status": "active",
  "strength": "strong",
  "description": "Test case verifies timing requirement compliance",
  "metadata": {
    "verification_method": "automated_test",
    "confidence": 0.95,
    "rationale": "Direct verification via timing measurement"
  },
  "suspect_reason": null,
  "created_by": "user_analyst123",
  "created_at": "2024-03-15T10:30:00Z",
  "updated_at": "2024-03-15T10:30:00Z"
}
```

### Relationship Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier (prefix: `link_`) |
| `project_id` | string | Parent project ID |
| `source_id` | string | Source item ID |
| `target_id` | string | Target item ID |
| `type` | string | Relationship type |
| `direction` | string | `forward`, `backward`, `bidirectional` |
| `status` | string | `active`, `suspect`, `obsolete` |
| `strength` | string | `strong`, `moderate`, `weak` |
| `description` | string | Optional description |
| `metadata` | object | Custom relationship data |
| `suspect_reason` | string | Reason if status is suspect |
| `created_by` | string | Creator user ID |
| `created_at` | string | ISO 8601 creation timestamp |
| `updated_at` | string | ISO 8601 last update timestamp |

### Relationship Types

| Type | Description | Direction | Example |
|------|-------------|-----------|---------|
| `satisfies` | Satisfies parent requirement | forward | Child req → Parent req |
| `derives_from` | Derived from another item | forward | Design → Requirement |
| `implements` | Implements specification | forward | Code → Design |
| `verified_by` | Verified by test | forward | Requirement → Test case |
| `validates` | Validates requirement | backward | Test case → Requirement |
| `traces_to` | General traceability | forward | Any → Any |
| `depends_on` | Dependency relationship | forward | Item → Dependency |
| `relates_to` | Related items | bidirectional | Item ↔ Related item |
| `duplicates` | Duplicate of another | forward | Item → Original |
| `blocks` | Blocks another item | forward | Blocker → Blocked |
| `conflicts_with` | Conflicts with another | bidirectional | Item ↔ Conflicting |
| `refines` | Refines/details another | forward | Detail → General |

### Relationship Status

| Status | Description |
|--------|-------------|
| `active` | Link is current and valid |
| `suspect` | Source or target changed; needs review |
| `obsolete` | Link no longer valid |

## Create Relationship

Create a traceability link between items.

```http
POST /v1/projects/{project_id}/links
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_id` | string | Yes | Source item ID |
| `target_id` | string | Yes | Target item ID |
| `type` | string | Yes | Relationship type |
| `description` | string | No | Link description |
| `strength` | string | No | Link strength |
| `metadata` | object | No | Custom link data |

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_abc123/links" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "source_id": "item_req_002",
    "target_id": "item_tc_005",
    "type": "verified_by",
    "description": "Test case verifies error handling requirement",
    "strength": "strong",
    "metadata": {
      "verification_method": "automated_test",
      "confidence": 0.95
    }
  }'
```

### Response

```json
{
  "data": {
    "id": "link_new123xyz",
    "project_id": "proj_abc123",
    "source_id": "item_req_002",
    "target_id": "item_tc_005",
    "type": "verified_by",
    "direction": "forward",
    "status": "active",
    "strength": "strong",
    "description": "Test case verifies error handling requirement",
    "metadata": {
      "verification_method": "automated_test",
      "confidence": 0.95
    },
    "created_by": "user_analyst123",
    "created_at": "2024-12-08T10:00:00Z"
  }
}
```

### SDK Examples

#### Python

```python
from tracertm import TraceRTMClient

client = TraceRTMClient(api_key="rtm_live_sk_...")

# Create simple link
link = client.links.create(
    project_id="proj_abc123",
    source_id="item_req_002",
    target_id="item_tc_005",
    type="verified_by"
)

print(f"Created link: {link.id}")

# Create with metadata
link = client.links.create(
    project_id="proj_abc123",
    source_id="item_req_003",
    target_id="item_design_001",
    type="implements",
    description="Design implements timing requirement",
    strength="strong",
    metadata={
        "verification_method": "inspection",
        "confidence": 0.9
    }
)
```

#### TypeScript

```typescript
import { TraceRTMClient } from '@tracertm/sdk';

const client = new TraceRTMClient({ apiKey: 'rtm_live_sk_...' });

// Create link
const link = await client.links.create({
  projectId: 'proj_abc123',
  sourceId: 'item_req_002',
  targetId: 'item_tc_005',
  type: 'verified_by',
  description: 'Test verifies error handling'
});

console.log(`Created link: ${link.id}`);
```

#### Go

```go
link, err := client.Links.Create(ctx, "proj_abc123", &tracertm.CreateLinkRequest{
    SourceID:    "item_req_002",
    TargetID:    "item_tc_005",
    Type:        "verified_by",
    Description: "Test verifies error handling",
})
if err != nil {
    log.Fatal(err)
}

fmt.Printf("Created link: %s\n", link.ID)
```

```http
GET /v1/projects/{project_id}/links
```

| `status` | string | `active` | Filter by status |
| `source_id` | string | — | Filter by source item |
| `target_id` | string | — | Filter by target item |
| `source_type` | string | — | Filter by source item type |
| `target_type` | string | — | Filter by target item type |
| `strength` | string | — | Filter by link strength |
| `suspect` | boolean | — | Only suspect links |
| `created_after` | string | — | Created after ISO date |
| `created_before` | string | — | Created before ISO date |
| `sort` | string | `created_at` | Sort field |
| `order` | string | `desc` | Sort order |
| `limit` | integer | 20 | Results per page (1-100) |

```bash
curl "https://api.tracertm.dev/v1/projects/proj_abc123/links?type=verified_by&status=active&limit=50" \
```

### SDK Examples

#### Python

```python
# List verification links
links = client.links.list(
    project_id="proj_abc123",
    type="verified_by",

)

    print(f"{link.source.item_number} → {link.target.item_number}")

suspect_links = client.links.list(
    project_id="proj_abc123",
    suspect=True
)

print(f"Found {len(suspect_links.data)} suspect links to review")

# Iterate through all links
for link in client.links.iterate(project_id="proj_abc123"):
    print(f"{link.type}: {link.source_id} → {link.target_id}")
```

## Update Relationship

Update a relationship's properties.

```http
PATCH /v1/links/{link_id}
```

### Request Body

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Change relationship type |
| `status` | string | Change status |
| `strength` | string | Change strength |
| `description` | string | Update description |
| `metadata` | object | Update metadata (merged) |

### Request

```bash
curl -X PATCH "https://api.tracertm.dev/v1/links/link_abc123" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "status": "active",
    "description": "Verified after source update",
    "metadata": {
      "verified_at": "2024-12-08T10:00:00Z",
      "verified_by": "user_analyst123"
    }
  }'
```

### SDK Examples

#### Python

```python
# Verify a suspect link
link = client.links.update(
    link_id="link_abc123",
    status="active",
    description="Verified after source update"
)

# Mark link as obsolete
link = client.links.update(
    link_id="link_old001",
    status="obsolete",
    metadata={"obsolete_reason": "Requirement deprecated"}
)
```

## Batch Create Relationships

Create multiple relationships in a single request.

```http
POST /v1/projects/{project_id}/links/batch
```

### Request Body

```json
{
  "links": [
    {
      "source_id": "item_req_001",
      "target_id": "item_tc_001",
      "type": "verified_by"
    },
    {
      "source_id": "item_req_001",
      "target_id": "item_tc_002",
      "type": "verified_by"
    },
    {
      "source_id": "item_req_002",
      "target_id": "item_design_001",
      "type": "implements"
    }
  ],
  "skip_duplicates": true
}
```

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_abc123/links/batch" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "links": [
      {"source_id": "item_req_001", "target_id": "item_tc_001", "type": "verified_by"},
      {"source_id": "item_req_001", "target_id": "item_tc_002", "type": "verified_by"},
      {"source_id": "item_req_002", "target_id": "item_tc_003", "type": "verified_by"}
    ],
    "skip_duplicates": true
  }'
```

### Response

```json
{
  "data": {
    "results": [
      {"index": 0, "success": true, "id": "link_new001"},
      {"index": 1, "success": true, "id": "link_new002"},
      {"index": 2, "success": false, "error": {"code": "LINK_EXISTS", "message": "Link already exists"}}
    ],
    "summary": {
      "total": 3,
      "created": 2,
      "skipped": 1,
      "failed": 0
    }
  }
}
```

### SDK Examples

#### Python

```python
results = client.links.batch_create(
    project_id="proj_abc123",
    links=[
        {"source_id": "item_req_001", "target_id": "item_tc_001", "type": "verified_by"},
        {"source_id": "item_req_001", "target_id": "item_tc_002", "type": "verified_by"},
        {"source_id": "item_req_002", "target_id": "item_tc_003", "type": "verified_by"}
    ],
    skip_duplicates=True
)

print(f"Created {results.summary.created}/{results.summary.total} links")
```

```bash
curl "https://api.tracertm.dev/v1/projects/proj_abc123/trace?from=item_req_001&to=item_tc_005&max_depth=5" \
```

#### Python

```python
trace = client.links.trace(
    project_id="proj_abc123",
    from_item="item_req_001",
    to_item="item_tc_005",
    max_depth=5
)

print(f"Found {trace.paths_found} paths")
for i, path in enumerate(trace.paths):
    print(f"\nPath {i+1} (length {path.length}):")
    for j, node in enumerate(path.nodes):
        print(f"  [{node['type']}] {node['item_number']}")
        if j < len(path.edges):
            print(f"    --{path.edges[j]['type']}-->")
```

```python
    item_id="item_req_001",
    direction="downstream",
    depth=5
)

print(f"Source: {impact.source_item.item_number}")
print(f"Total impacted: {impact.summary.total_impacted}")
print(f"\nBy type:")
for item_type, count in impact.summary.by_type.items():
    print(f"  {item_type}: {count}")

print(f"\nRecommendations:")
for rec in impact.recommendations:
    print(f"  [{rec.priority}] {rec.action}: {rec.description}")
```

```json
{
  "error": {
    "code": "LINK_EXISTS",

    "details": {

    }
  }
```

### Self-Link

```json
{
  "error": {
    "code": "INVALID_LINK",
    "message": "Cannot create a link from an item to itself",
    "status": 422
  }
}
```

