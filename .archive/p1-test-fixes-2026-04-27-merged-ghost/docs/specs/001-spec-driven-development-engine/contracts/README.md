## How regeneration works

The `.github/workflows/regen-docs-specs.yml` workflow runs on push to `main`
whenever `proto/**` changes. It copies every `*.proto` from
`proto/agileplus/v1/` into this directory, verbatim. Hand-maintained files
listed above are never touched.

Commits are attributed to `agileplus-proto-sync[bot]` with `[skip ci]` to avoid
recursion. See SSOT audit issue #121 step 3 and Tracera PR #342 for the pattern.

## Do not edit auto-regenerated files here

Edit the canonical `proto/agileplus/v1/*.proto` files instead. CI will sync.
