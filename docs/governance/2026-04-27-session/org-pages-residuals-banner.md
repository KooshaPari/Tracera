# Org-Pages Residual Banner Spec

**Goal:** Make dependency-advisory state visible at a glance across phenotype-org's portfolio dashboard.

## Visual
```

  Last full sweep: 2026-04-27 (Phenotype-org)
  Source: dependabot-alerts API; refreshed every 6h
```

## Implementation
1. **Generator script** (Vercel cron, every 6h):
   ```bash
   gh api repos/<r>/dependabot/alerts --paginate \
     --jq '[.[] | select(.state=="open")] | group_by(.security_advisory.severity) | ...'
   ```
2. **Output**: `state.json` posted to org-pages site (CF Pages or Vercel).
3. **Banner component**: React/HTML, reads `state.json`, renders with sparklines.
4. **Ownership**: dashboard owner runs cron; phenotype-org infra cluster.

## Acceptance criteria
- [ ] Refreshes every 6h without GH-rate-limit-spam
- [ ] Renders correctly on mobile + desktop
- [ ] Single GraphQL query (not N×K loops)
- [ ] Cached at edge (Cloudflare) for <100ms TTFB

## Dependencies on residuals state
- `repos/docs/governance/2026-04-27-session/residuals-state.json` is the seed dataset
- Will be auto-regenerated by the cron once deployed

## Anti-patterns
- ❌ Real-time GH polling on page load (will hit rate limit)
- ❌ Static + manual refresh (drifts)
- ❌ Showing arbitrary "score" instead of raw counts (loses signal)
